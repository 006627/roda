Bem-vindo ao RODA!

Um repositório open-source projetado para a preservação digital

O RODA é uma solução que oferece funcionalidades para todas as principais unidades funcionais do modelo de referência OAIS. Este software é capaz de incorporar, gerir e dar acesso aos vários tipos de material digital produzido no âmbito das atividades de grandes organizações ou entidades públicas. O RODA é baseado em tecnologias open-source e é suportado por normas já existentes, tais como o Open Archival Information System (OAIS), Metadata Encoding and Transmission Standard (METS), Encoded Archival Description (EAD), Dublin Core (DC) e Metadados de preservação (PREMIS).

Suportado por normas abertas

O RODA é suportado por normas abertas utilizando o EAD como esquema de metadados de descrição, PREMIS para metadados de preservação, METS para metadados estruturais e várias normas para metadados técnicos (por exemplo, NISO Z39.87 para imagens fixas digitais).

Independente do fornecedor

O RODA é 100% baseado em tecnologias open-source. Toda a infraestrutura necessária para suportar o RODA é, também esta, independente do fornecedor. Isso significa que se podem usar as distribuições Linux que melhor se adequam às suas necessidades institucionais.

Escalável

A natureza orientada ao serviço do RODA permite que o sistema seja altamente escalável, permitindo a distribuição da carga de processamento entre vários servidores. Além disso, o uso de componentes avançados de indexação permite que os serviços de descoberta do RODA sejam distribuídos por vários servidores num cluster para um desempenho ainda maior.

Ações de preservação incorporadas

As ações de preservação e gestão no interior do RODA são desempenhadas por um módulo de execução de tarefas. O módulo de execução de tarefas permit ao gestor do repositório executar tarefas de preservação num dado conjunto de dados de forma paralela para maior eficiência. As ações de preservação incluem conversões de formatos, verificações de integridade, comunicação, despiste de vírus, etc.

Autenticidade

O RODA usa metadados de preservação (PREMIS) para criar uma cadeia de confiança entre todas as transformações realizadas ao conteúdo ingerido. Os metadados de preservação, em conjunto com o estabelecimento da confiança do meio envolvente (ISO 16363), asseguram a fiabilidade do serviço e a autenticidade dos objetos digitais incorporados.

Suporte para vários formatos

O RODA é capaz de ingerir todo o tipo de conteúdo. Os plugins de preservação suportam a migração de documentos de texto, imagens , bases de dados relacionais, vídeo e áudio para formatos normalizados para preservação a longo prazo. Um mecanismo de plugins permite que o RODA suporte facilmente migrações de formatos adicionais.

Capaz de lidar com a rápida mudança da tecnologia

O motor de execução de tarefas disponibiliza uma maneira simples de adicionar mais funcionalidade ao sistema (por exemplo, novas ações de preservação, alertas, ferramentas, etc.). Além disso, a arquitetura orientada ao serviço do RODA permite que os seus componentes sejam progressivamente atualizados, permitindo um uso heterogêneo de tecnologias a todos os níveis: hardware, sistema operativo e aplicações.

Controlo de acessos avançado

Todos os utilizadores têm de estar autenticados para poderem aceder ao repositório. Todas as ações do utilizador são registadas para responsabilização futura. As permissões são granulares e podem ser definidas desde o nível do repositório até ao nível dos objetos de dados individuais.

Interoperabolidade

O RODA expõe todas as suas funcionalidades através de uma API REST bem documentada. Bibliotecas Java encontram-se, também, disponíveis no GitHub para permitir que os programadores interajam com o RODA através das suas APIs. Várias ferramentas existem para criar e manipular SIPs e submetê-los ao workflow de ingestão do RODA.

Workflow de ingestão avançado

O RODA suporta a ingestão de novos materiais digitais, bem como metadados associados em vários formatos distintos. São fornecidas ferramentas para permitir que os Produtores criem SIPs nos formatos suportados. O workflow de ingestão pode ser personalizado na interface do utilizador para lidar com políticas e incorporação particulares.

E muito mais...

O RODA é um repositório digital para preservação a longo prazo que implementa um workflow de ingestão que não só valida SIPs, mas também realiza identificação de formatos, extrai metadados técnicos e migra os objetos digitais para formatos mais "preserváveis".

O RODA também dá acesso a informações digitais de várias formas, tais como navegação e pesquisa nos metadados, assim como no conteúdo dos ficheiros, visualização em-linha e download de originais e derivadas.

As interfaces de administração permitem que os utilizadores do backoffice do repositório façam a gestão dos fundos/coleções e definam regras para as ações de preservação. Todas as interações entre os utilizadores (humanos e máquinas) e o repositório são registadas por razões de segurança e responsabilização.

O RODA garante que os dados ingeridos são autênticos através do registo de metadados PREMIS sobre todas as ações executadas pelo repositório, registando a proveniênciados dos dados e assegurando a sua integridade através da monitorização permanente dos dados de modo a garantir que estes não foram adulterados.
